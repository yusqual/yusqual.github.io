总结几种去重方法

  1. if (i > 0 && nums[i] < nums[i - 1]) continue; 用于对递归方向进行去重
  2. if (i > startIndex && nums[i] < nums[i - 1]) continue; 用于对当前层进行去重(题目对元素顺序无要求, 需要先排序)
  3. 每层定义一个unordered_ 对当前层进行去重. if (uset.find(nums[i]) != uset.end()) continue; (题目对元素顺序有要求, 不能排序)

类似于n=4, k=4的情况, 在第一层的for循环中, 当i取到2以及以后的数时, 递归不论怎么取数都取不到k=4个数, 这部分递归可以剪枝:

  1. 已经选择的元素个数: path.size()
  2. 还需的元素个数: k - path.size()
  3. 列表中剩余元素个数(n - i) >= 还需的元素个数(k - path.size())
  4. 至多起始位置: i <= n - (k - path.size()) + 1

332. 重新安排行程

题目:

给你一份航线列表 tickets ，其中 tickets[i] = [fromi, toi] 表示飞机出发和降落的机场地点。请你对该行程进行重新规划排序。
所有这些机票都属于一个从 JFK（肯尼迪国际机场）出发的先生，所以该行程必须从 JFK 开始。如果存在多种有效的行程，请你按字典排序返回最小的行程组合。
例如，行程 [“JFK”, “LGA”] 与 [“JFK”, “LGB”] 相比就更小，排序更靠前。
假定所有机票至少存在一种合理的行程。且所有的机票 必须都用一次 且 只能用一次。

思路

假设使用 vector<string> path 来记录路径, 首先想到的就是根据 path.back() 也就是上一个机场, 到 tickets 中去寻找 tickets[0] 也就是出发地相同的票, 然后将目的地 tickets[1] 添加到 path 中, 再递归地去查找. 当 path.size() 等于 tickets.size() + 1 时, 就说明找到了一个结果, 因为 path 是站台, 而每个 tickets 是站台到站台, 所以需要+1. 代码如下:

很明显, 由于查找了所有可能的结果, 时间复杂度过高.
可以在最开始把tickets进行排序, 这样第一次找到的结果就是最终结果, 并且将backtracking的返回值设为 bool , 当返回值为 true 时直接返回上层, 不再继续寻找.
