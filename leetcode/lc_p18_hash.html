242. 有效的字母异位词

题目:

给定两个字符串 s 和 t ，编写一个函数来判断 t 是否是 s 的 字母异位词.

思路

创建一个char[26]的数组, 遍历s时把对应元素++, 遍历t时对应元素–; 最后检查所有元素是否均为0.

202. 快乐数

题目:

编写一个算法来判断一个数 n 是不是快乐数。
「快乐数」 定义为：

  • 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
  • 然后重复这个过程直到这个数变为 1，也可能是 无限循环 但始终变不到 1。
  • 如果这个过程 结果为 1，那么这个数就是快乐数。

如果 n 是 快乐数 就返回 true ；不是，则返回 false 。

思路

使用uset存储每次出现的sum, 如果在存储前发现uset中已经存在了当前sum, 则可判断为出现无限循环, 直接返回false;
如果sum为1,则返回true.

1. 两数之和

题目:

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。
你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。
你可以按任意顺序返回答案。

思路

遍历一次nums数组, 每次遍历从uset中寻找是否存在一个值, 使得这个值和当前数之和等于target,
若存在, 则返回这两个数, 若不存在, 则把 target-nums[i] 存入uset.

454. 四数相加 II

题目:

给你四个整数数组 nums1、nums2、nums3 和 nums4 ，数组长度都是 n ，请你计算有多少个元组 (i, j, k, l) 能满足：

  • 0 <= i, j, k, l < n
  • nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0

思路

使用两个双层for循环, 第一个遍历nums1 nums2排列出来的和的可能, 存到umap<int, int>里(两数的和, 出现次数);
第二个遍历nums3 nums4排列出来的和的可能, 再从umap中寻找是否存在某个值使得相加得0.

15. 三数之和

题目:

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请你返回所有和为 0 且不重复的三元组。
注意：答案中不可以包含重复的三元组。

思路

这题最好不要使用哈希的方法, 涉及去重、剪枝都很麻烦.
可以采用双指针的方法, 先将数组进行排序, 用一个fori循环从左到右遍历, 再令左指针为i+1, 右指针为size-1, 判断i left right对应的nums元素之和与0的大小关系, 小了就右移left, 大了就左移right.
