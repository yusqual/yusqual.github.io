1. 架构与加载流程

TAS Kit 通过 L4D2 的 Scripted Mode 系统启动。当游戏加载一个拥有自定义 scripts/vscripts/ 的地图时，引擎调用以下入口：

关键：lib_utils.nut 被最先加载，因为它定义了所有后续代码依赖的常量、枚举、类和函数。tas_kit.nut 加载 9 个模块，最后加载用户的 main.nut。

2. IScriptPlugin 接口

CTASKit 继承 IScriptPlugin（定义在 lib_utils.nut）。接口定义：

方法 说明
Load() 插件加载时调用。设置 Cvar、加载模块、注册事件
Unload() 插件卸载时调用
OnRoundStartPost() 回合开始后调用
OnRoundEnd() 回合结束时调用
AdditionalClassMethodsInjected() CTerrorPlayer 扩展方法注入后调用
GetClassName() 返回类名 "CTASKit"
GetScriptPluginName() 返回插件名 "TAS Kit"
GetInterfaceVersion() 返回接口版本号 1

插件通过 g_ScriptPluginsHelper.AddScriptPlugin(g_TASKit) 注册（在 tas_kit.nut 末尾）。

3. Speedrun 系统

3.1 全局状态

变量 类型 说明
g_bRestarting bool 是否正在执行重启流程（初始 false）
g_bSpeedrunStarted bool 计时器是否已启动
g_bApplyMapParams bool 是否应用地图参数（初始 true）
g_flFinaleStartTime float | null 终局开始时间戳
g_Timer table 计时数据 {start_time, time, previous_segment, HUD}
g_tSegmentData table 按角色名索引的分段数据 {nick=[data], ...}
Players table 角色显示名 → 玩家实体映射

3.2 StartSpeedrun

清除重启标志，设置速通运行中标志，记录起始时间戳。由 Countdown(0) 调用。

3.3 RestartSpeedrun

流程：

  1. 设置 g_bRestarting = true（下次 Load() 会检测到此标志并加载 main.nut）
  2. 重启游戏回合（mp_restartgame 1）
  3. 重置时间流速为 1x
  4. 禁用换关触发器（防止意外换关）
  5. 触发 OnSpeedrunRestart 回调

3.4 PrintTime

在终局撤离载具离开、地图过渡、玩家死亡/断线时触发。输出三种时间格式：

  • 终局事件时间（如果有终局起点）
  • 总时间（previous_segment + time）
  • 分段计时（当前分段时间）

超过 60 秒时附带 MM:SS 格式。

3.5 OnCheckpointDoorClosed

安全门关闭或玩家踩到 info_changelevel 触发器时，触发引擎关卡过渡逻辑。

3.6 InitializeHooks

在重启后重新建立所有事件钩子和实体 I/O 钩子（因为 mp_restartgame 会清除之前的钩子）。

4. HUD 计时器

4.1 InitializeHUD

创建三个 HUD 字段：

  • timer_sec：左上，右对齐，显示为时间格式（M:SS）
  • timer_ms：左中，无背景，显示毫秒（000）
  • timer_comma：中间上方，无背景，分隔符（'）

如果 g_tDataTable.hud == false，则设置 HUD_FLAG_NOTVISIBLE。

4.2 HUD_Think

每帧更新 HUD 显示。仅在 g_tDataTable.hud == true 且 g_bSpeedrunStarted == true 时工作。

5. 倒计时系统

5.1 Countdown

第一次调用时（iValue == 3）：

  1. 调用 Countdown_Think() 冻结所有幸存者并清除物品栏
  2. 注册 Countdown_Think 为 OnTick 函数
  3. 按 timer/3 间隔依次创建定时器调用 Countdown(2), Countdown(1), Countdown(0)

Countdown(0) 时：

  1. 移除 Countdown_Think
  2. 调用 StartSpeedrun()
  3. 解冻幸存者，禁用 LedgeHang
  4. 为每个幸存者调用 IssueSurvivorEquipment()
  5. 触发 OnSpeedrunStart 回调

5.2 Countdown_Think

在倒计时期间每帧执行：

  • 冻结所有幸存者（设置 FL_FROZEN）
  • 清除物品栏（防止玩家在倒计时期间换武器）

6. 幸存者装备

6.1 IssueSurvivorEquipment

完整的幸存者状态恢复：

  1. TP(hPlayer, vecPos, eAngles, vecVel) — 传送
  2. hPlayer.SetHealth(iHealth) — 设置血量
  3. hPlayer.SetHealthBuffer(flHealthBuffer) — 设置虚血
  4. hPlayer.SetReviveCount(iRevivies) — 设置倒地次数
  5. 遍历 aInventory[0] 发放武器（hPlayer.GiveItem()）
  6. 切换到活跃武器（aInventory[1]）
  7. 应用武器升级（aInventory[4]）
  8. 设置弹药（hPlayer.SetAmmo() — 主武器弹夹/备弹/升级弹药，副武器弹夹）
  9. 执行 funcActions(hPlayer) — 用户自定义操作

6.2 g_tSegmentData 与 Players 表

支持全部 8 名幸存者（Coach, Ellis, Nick, Rochelle, Louis, Zoey, Francis, Bill）。

7. 地图参数 (Map Params)

InitializeMapParams() 为以下地图消除 RNG（仅在 g_bApplyMapParams == true 时执行）：

地图 处理的 RNG
c1m3_mall 移除中间随机路径（minifinale_trigger）、移除随机扶梯路径（escalator_trigger）
c2m4_barns 移除随机栅栏（randompath_1_*）
c2m5_concert 使用左侧直升机撤离路线（快约 2 秒）
c5m3_cemetery 移除随机墓地路径（case_maze_config），固定使用 Path_C
c6m1_riverbank 预缓存 Witch 模型
c7m1_docks 自定义开场：生成 3 只带胆汁的 CEDA 感染者 + 7 只普通感染者，便于获取胆汁

8. 数据持久化

读写流程：

  1. 加载时：从 ems/tas_kit/data.nut 读取 → 合并到 g_tDataTable（已有键优先）
  2. 写入时：调用 UpdateDataTable(g_tDataTable) → 序列化为 { key = "value" } 格式 → 写入文件
