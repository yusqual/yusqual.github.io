1. tools

文件：modules/tools.nut（~1760 行）

最核心的功能模块，提供物品/僵尸/Trigger 生成、Common Boost、自动拾取/交互、Auto Boost（火箭跳/胆汁跳/手雷跳）、自动攀爬、CI 录制等功能。

1.1 全局数据表

g_tItems

武器/物品/道具的完整定义表。键为短名称，值为 { classname, model }。

包含全部 L4D2 武器、近战、投掷物、药品和物理道具的定义。

g_tTransitionLandmarks

地图过渡坐标数据，用于跨地图的坐标转换。每张地图的条目包含 [当前图原点X, Y, Z, 下一图原点X, Y, Z]。

其他

变量 说明
g_aPrimaryWeaponList 所有主武器 classname 数组
g_aPropPhysicsModels 可携带物理道具模型路径数组
g_aUseEntitiesList 可交互实体 classname 数组（门、升级台、按钮）
g_aBilesToProcess 待自动打爆的胆汁抛射物队列

1.3 Common Boost 系统

利用普通感染者攻击动作来辅助加速的系统。

工作流程：

  1. SpawnCommonForCB() 在指定位置生成闲置感染者并注册到 tCB.aList
  2. 玩家开火时 OnWeaponFire 遍历 tCB.aList，按距离匹配最近的感染者
  3. tCB.TriggerCI() 设置 m_hTarget 为指定玩家，感染者自动寻路攻击

2. tls

文件：modules/tls.nut（~840 行）

TLS = “Totally Legit Scripts”。提供 Aimbot（自动瞄准）、Ragebot（自动开火瞄准）和 Auto-Bhop（自动连跳）功能。

2.1 ConVars

TLS 通过以下 ConVar 控制行为（使用 CreateConVar 创建）：

ConVar 类型 默认值 说明
tls_aimbot int 1 Aimbot 总开关（0 禁用全部）
tls_aimbot_radius float 3000.0 瞄准最大距离
tls_aimbot_emit_shoot_sound int 1 是否发射射击音效
tls_aimbot_ignore_obstacles int 0 是否穿墙瞄准
tls_aim_anytime int 0 是否允许随时瞄准（不受武器冷却限制）
tls_aim_to_head int 1 是否瞄准头部（0 = 身体）
tls_ragebot int 1 Ragebot 总开关
tls_autobhop int 1 Auto-Bhop 总开关

2.2 辅助函数

2.3 Aimbot 系统

Aimbot — 自动将视角对准最近的感染者。

Aimbot 工作模式：

  1. 检测到瞄准目标 → 计算头部/身体位置
  2. SetAnglesToNearestEntity(hPlayer) → 转动视角
  3. EmitShootSound() → 发送射击音效
  4. hPlayer.SendInput(IN_ATTACK) → 按下攻击键

2.4 Ragebot 系统

Ragebot — 在 TLS_Think 中每帧自动检测目标并瞄准开火。

Ragebot 与 Aimbot 的区别：Ragebot 不需要按键触发，每帧自动执行。

2.5 Auto-Bhop 系统

Auto-Bhop — 在落地瞬间自动跳跃。

4. finale_manager

文件：modules/finale_manager.nut（~283 行）

终局增强层——在默认导演脚本刷怪的基础上，将生成的 CI 重新放置到预设位置，并可跟踪 Tank 击杀数与阶段耗时。

重要：finale_manager 不会屏蔽或替换游戏的默认导演脚本。地图自带的导演脚本仍然控制阶段推进（Panic → Tank → Delay → …）和刷怪节奏。finale_manager 只在 CI 被刷出后将其「搬运」到用户预设的坐标，本质是被动的增强层，不是导演系统的替代品。如果需要在终局完全自定义刷怪流程（跳过阶段、改变 CI 数量等），还需配合 director_no_mobs 1 等 Cvar 手动屏蔽默认导演，并用 Z_Spawn() 自行控制生成。

7. autosb

文件：modules/autosb.nut（~190 行）

自动 Spitterboost——当 bot Spitter 喷射酸液时，自动计算弹跳时间并跳跃。

函数

工作原理

AutoSB_Think() 每帧检查：

  1. 是否有 bot Spitter 激活了 spit 技能（检测 m_flSpecialStartTime 变化）
  2. 计算 Spitter 与目标玩家的距离
  3. 根据距离计算最佳跳跃时间：jumpTime = 0.333374 + (distance / 900) - (0.63333 * factor)
  4. 在计算的时间点自动发送 IN_JUMP

全局状态

8. fillbot

文件：modules/fillbot.nut（~145 行）

自动灌油机器人——用于 scavenge 终局中自动拾取油桶并浇入发电机。

函数

工作原理

FillBot_Think(hPlayer) 作为 per-player tick 运行：

  1. 如果未持有油桶 → 查找 105 单位半径内最近的 weapon_gascan → 自动拾取
  2. 如果持有油桶但未浇油 → 查找 65 单位半径内最近的 point_prop_use_target → 转向 + StartPouring()
  3. 检测浇油完成/中断/受阻事件 → StopPouring()

注册的事件钩子

  • item_pickup → 如果拾取的是油桶，自动开始浇油
  • adrenaline_used → 关闭 Fillbot
  • gascan_pour_completed → 停止浇油
  • gascan_pour_interrupted → 停止浇油
  • gascan_pour_blocked → 停止浇油

跳过所有官方战役的开场动画。

函数

地图 跳过的内容

c2m1_highway LCS 过场、fade、点视角
c3m1_plankcountry LCS 过场、fade、点视角
c4m1_milltown_a 拖船动画、LCS 过场、fade

c6m1_riverbank LCS 过场、fade
c7m1_docks LCS 过场、fade
c8m1_apartment LCS 过场、fade
c9m1_alleys LCS 过场、fade
c10m1_caves LCS 过场、fade
c11m1_greenhouse 拖船、fade、点视角
c12m1_hilltop LCS 过场、fade
c13m1_alpinecreek LCS 过场、fade
uf1_boulevard 飞机残骸

实现方式

通过 EntFire() Kill 过场相关的实体（lcs_intro, fade_intro, point_viewcontrol_survivor 等），并在 0.1 秒后调用 director.FinishIntro() 和 director.ReleaseSurvivorPositions()。

SkipIntro() 是通用回退方案，当 g_bBeginMap == true 但地图不在上述列表中时使用。
