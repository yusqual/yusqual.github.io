1. 前置要求

TAS Kit 依赖以下 SourceMod 插件：

  • Speedrunner Tools (v1.4+) — 提供 sm_idle、sm_fake、sm_ccmd、sm_name 等命令的底层实现
  • Movement Reader (v1.0+) — 可选，sm_mr_* 系列命令需要此插件

此外需要确保以下 Cvar 可在 VScript 中设置：

  • sv_cheats 1
  • sb_all_bot_game 1

TAS Kit 在加载时自动设置这些值。

3. 架构概览

TAS Kit 通过 L4D2 的 Scripted Mode（脚本化模式）系统启动。加载流程如下：

关键点：

  • scriptedmode.nut 是引擎级入口，设置 g_bOverrideGameHooks = true
  • lib_utils.nut 提供 IScriptPlugin 接口和所有底层工具
  • tas_kit.nut 的 CTASKit 类实现了插件接口，负责加载 9 个模块
  • 所有模块共享全局作用域（通过 IncludeScript 加载到 roottable）

4. 第一个速通脚本

在你的地图对应 VScript 目录下创建 main.nut。以下是一个最小工作示例：

运行流程：

  1. 首次加载地图 → CTASKit::Load() 执行 → 应用 Cvars、设置 sv_cheats 1 等
  2. 输入 !restart 聊天命令 → RestartSpeedrun() → mp_restartgame 1
  3. 重启后重新执行 Load() → 检测到 g_bRestarting → 加载 main.nut
  4. 应用 Cvars、恢复 Survivors 状态 → 倒计时 3-2-1
  5. 倒计时归零 → 解冻玩家、发放装备 → 调用 OnSpeedrunStart
  6. HUD 计时器开始运行

5. 模块加载顺序

LoadModules() 按以下顺序加载模块。顺序影响依赖关系：

顺序 模块 依赖
1 speedrunner_tools 依赖 lib_utils.nut
2 helper_utils 依赖 speedrunner_tools（需要 SessionState.MapName）
3 skipintro 独立（仅使用游戏 API）
4 tls 依赖 lib_utils.nut
5 autosb 依赖 lib_utils.nut
6 fillbot 依赖 lib_utils.nut
7 tools 依赖 helper_utils、speedrunner_tools
8 finale_manager 依赖 tools（tScriptedTankSpawn）
9 chat_commands 依赖以上所有模块的函数

6. 约定

命名约定

  • 速通脚本命名为 main.nut，放在地图对应的 scripts/vscripts/ 目录下
  • 数据文件自动写入 ems/tas_kit/data.nut

回调命名

  • ::Callbacks 的回调是可选的可选函数
  • ::SetCameraAnglesCompleted 是直接注册在根表的全局函数

数据表持久化

  • g_tDataTable 中的值在重启间保持
  • 使用 UpdateDataTable() 写入文件

7. 数据持久化

TAS Kit 将用户配置保存到 left4dead2/ems/tas_kit/data.nut，在重启/重载间保持。

在 main.nut 中修改配置

通过聊天命令修改

命令 功能
!timer 5 设置倒计时为 5 秒
!fdmg 切换摔伤报告
!hud 切换 HUD 计时器显示
